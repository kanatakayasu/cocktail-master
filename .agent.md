# エージェントチーム構成

このプロジェクトで使用するエージェントチームの定義。

## チーム構成

### 🎨 ui-designer
**役割**: UIコンポーネント・ページデザインの実装
**担当範囲**:
- React コンポーネント設計・実装
- Tailwind CSS スタイリング
- Framer Motion アニメーション
- レスポンシブデザイン
- アクセシビリティ対応

### 📊 data-architect
**役割**: データ層の設計・実装
**担当範囲**:
- TypeScript 型定義
- 学習データ（カクテル、スピリッツ、クイズ等）の構造化
- データバリデーション
- LocalStorage による進捗保存ロジック

### 🧪 quiz-engineer
**役割**: クイズ・学習機能の実装
**担当範囲**:
- クイズロジック（出題・採点・解説）
- フラッシュカード機能
- 学習進捗トラッキング
- スペースドリピティション（間隔反復）ロジック

### 🔍 researcher
**役割**: 学習コンテンツの調査・検証
**担当範囲**:
- カクテルレシピの正確性検証
- 資格試験の出題傾向調査
- 新しいコンテンツの追加提案
- docs/ 配下のドキュメント管理

### 🛠 devops
**役割**: ビルド・テスト・デプロイ
**担当範囲**:
- Vite ビルド設定
- ESLint / TypeScript 設定
- テスト環境構築
- デプロイ設定

## 並行作業ガイドライン

1. **データ層** と **UI層** は独立して並行開発可能
2. 各ページ（図鑑・クイズ・暗記カード・分類）は独立したファイルなので並行開発可能
3. 共通コンポーネント (`src/components/`) の変更は他エージェントと調整が必要
4. `src/types/index.ts` は全エージェント共有のインターフェース定義（先に確定させる）

## ワークフロー

```
1. researcher → 学習コンテンツ収集・検証
2. data-architect → 型定義・データ構造化（types/ + data/）
3. ui-designer + quiz-engineer → ページ・機能実装（並行）
4. devops → ビルド確認・最適化
```
